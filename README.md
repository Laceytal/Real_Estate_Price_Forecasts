# Анализ финансовых транзакций и клиентов

Аналитический проект по обработке, очистке и исследованию данных финансовых транзакций и клиентских профилей консалтинговой компании. Проект включает полный цикл работы с данными: от загрузки и очистки до статистического анализа, объединения таблиц и визуализации результатов.



## Описание данных

### transactions_data.xlsx

Набор данных о финансовых транзакциях, содержащий 10 000 записей со следующими полями:

| Поле             | Тип       | Описание                                 |
|------------------|-----------|------------------------------------------|
| `transaction_id` | UUID      | Уникальный идентификатор транзакции      |
| `client_id`      | UUID      | Идентификатор клиента (может быть пустым)|
| `transaction_date`| datetime | Дата и время совершения транзакции       |
| `service`        | string    | Наименование финансовой услуги           |
| `amount`         | float     | Сумма транзакции                         |
| `payment_method` | string    | Способ оплаты                            |
| `city`           | string    | Город клиента                            |
| `consultant`     | string    | Имя консультанта                         |

Доступные услуги: Инвестиционное консультирование, Управление активами, Налоговое планирование, Финансовое планирование, Структурирование капитала.

### clients_data.json

Набор данных о клиентах, содержащий записи со следующими полями:

| Поле       | Тип    | Описание                              |
|------------|--------|---------------------------------------|
| `id`       | UUID   | Уникальный идентификатор клиента     |
| `age`      | float  | Возраст клиента (может быть пустым)   |
| `gender`   | string | Пол клиента (может быть пустым)       |
| `net_worth`| float  | Чистый капитал клиента (может быть пустым) |

## Этапы анализа

### 1. Очистка и подготовка данных

На этом этапе исходные данные проходят базовую проверку и фильтрацию:

- **Транзакции**: удаление строк с отсутствующим `transaction_id`, фильтрация отрицательных и нулевых сумм (`amount > 0`), приведение дат к единому формату `datetime`. После очистки остаётся **9 089 записей**.
- **Клиенты**: удаление записей без `id`, фильтрация отрицательных значений `net_worth`. После очистки остаётся **9 323 записи**.
- Данные содержат пропущенные значения в полях `client_id`, `age`, `gender`, `net_worth`, а также маркеры неизвестных значений ("Неизвестно", "Неизвестная услуга", "Неизвестный город"). Фильтрация выполняется по каждой задаче индивидуально, чтобы сохранить максимальный объём данных для каждого расчёта.

### 3. Объединение данных и сегментация клиентов

Транзакции объединены с профилями клиентов по полю `client_id` (LEFT JOIN). Создана категориальная переменная `net_worth_level` на основе чистого капитала:

| Категория         | Условие          |
|--------------------|------------------|
| Низкий капитал     | `net_worth < 100 000`    |
| Средний капитал    | `100 000 <= net_worth < 1 000 000` |
| Высокий капитал    | `net_worth >= 1 000 000` |

**Результат**: наибольшую выручку приносит категория клиентов с высоким капиталом.

### 4. Визуализация данных (дополнительное задание)

Построены три графика:

1. **Распределение сумм транзакций** -- гистограмма с ядром оценки плотности (KDE) показывает распределение сумм всех транзакций.
2. **Диаграмма выручки по услугам** -- столбчатая диаграмма, визуализирующая суммарную выручку для каждой финансовой услуги.
3. **Зависимость средней суммы транзакции от возраста** -- линейный график с линией тренда, построенной с помощью `LinearRegression` из scikit-learn, демонстрирующий, как средний размер транзакции меняется в зависимости от возраста клиента.

### 5. Прогнозирование спроса (дополнительное задание)

Прогнозирование количества заказов по каждой услуге на следующий месяц с использованием линейной регрессии (`LinearRegression` из scikit-learn).

**Подготовка данных:**

- Из объединённого датасета `df` удаляются записи с неизвестной услугой ("Неизвестная услуга").
- Создаётся колонка `month_year` с помощью `dt.to_period('M')` для группировки по месяцам.
- Данные агрегируются: для каждой пары (месяц, услуга) подсчитывается количество транзакций (`count`).
- Извлекается номер месяца (`month`) в качестве числового признака. Поскольку данные охватывают один год, для идентификации достаточно номера месяца; при наличии данных за несколько лет потребовалось бы дополнительное кодирование года.
- Категориальный признак `service` преобразуется в числовые столбцы с помощью One-Hot Encoding (`pd.get_dummies`).

**Формирование прогнозных данных:**

- Создаётся датафрейм для следующего месяца (месяц, следующий за максимальным в данных).
- Для каждой услуги генерируется строка с соответствующим номером месяца.
- Применяется то же One-Hot Encoding, а метод `reindex` гарантирует совпадение столбцов с обучающей выборкой.

**Обучение модели и прогноз:**

```python
model = LinearRegression()
X = monthly_revenue_encoded[X_cols]    # признаки: номер месяца + one-hot услуги
y = monthly_revenue_encoded['count']    # целевая переменная: количество транзакций
model.fit(X, y)
future_count = model.predict(future_month_encoded[X_cols])
```

Модель обучается на месячной динамике количества заказов по каждой услуге и прогнозирует ожидаемое число транзакций на следующий месяц. Результаты сортируются по убыванию прогнозного значения, что позволяет определить, какие услуги ожидается наиболее востребованными.
